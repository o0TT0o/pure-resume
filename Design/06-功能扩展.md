# 06-功能扩展

## 功能扩展指南

本文档说明如何在当前框架下扩展简历功能，以添加新的模块为例进行说明。

---

## 扩展思路

系统采用**配置驱动架构**，添加新模块无需修改代码，只需编辑配置文件和数据文件。

---

## 示例：添加"同事评价"模块

### 实施步骤（按顺序）

#### 步骤 1: 定义模块配置

**文件**: `data/modules/config.json`

```json
"testimonials": {
  "id": "testimonials",
  "i18nKey": "module.testimonials",
  "required": false,
  "file": "testimonials.json",
  "type": "array"
}
```

**字段说明**:
- `id`: 模块唯一标识 (小写字母+连字符)
- `i18nKey`: 国际化键值
- `required`: 是否必填 (false表示可选)
- `file`: 数据文件名
- `type`: 数据类型 (array表示多条记录)

---

#### 步骤 2: 定义列表字段配置

**文件**: `data/lists/config.json`

```json
"testimonials": {
  "id": "testimonials",
  "i18nKey": "list.testimonials.name",
  "fields": [
    {"id": "name", "i18nKey": "list.testimonials.field.name", "required": true},
    {"id": "position", "i18nKey": "list.testimonials.field.position", "required": true},
    {"id": "company", "i18nKey": "list.testimonials.field.company", "required": false},
    {"id": "relationship", "i18nKey": "list.testimonials.field.relationship", "required": false},
    {"id": "date", "i18nKey": "list.testimonials.field.date", "type": "date", "required": false},
    {"id": "content", "i18nKey": "list.testimonials.field.content", "type": "textarea", "required": true}
  ]
}
```

**字段说明**:
- `name`: 评价人姓名 (必填)
- `position`: 评价人职位 (必填)
- `company`: 评价人公司 (可选)
- `relationship`: 关系描述 (如"前同事"、"上级")
- `date`: 评价日期 (可选)
- `content`: 评价内容 (必填, textarea类型)

---

#### 步骤 3: 创建数据文件

**文件**: `data/zh-CN/testimonials.json`

```json
[
  {
    "id": "testimonial-001",
    "name": "张三",
    "position": "高级工程师",
    "company": "某某科技公司",
    "relationship": "前团队同事",
    "date": "2025-01",
    "content": "与他共事两年，技术能力强，团队协作出色，是项目成功的关键人物。"
  }
]
```

**注意**: 需要在 `en-US/`, `de-DE/`, `fr-FR/`, `ja-JP/` 目录下分别创建对应的 `testimonials.json`

---

#### 步骤 4: 添加国际化配置

**文件**: `languages/zh-CN/config-i18n.json`

```json
{
  "module": {
    "personal": "个人信息",
    ...
    "testimonials": "同事评价"
  },
  "list": {
    "experience": {
      ...
    },
    "testimonials": {
      "name": "同事评价",
      "field": {
        "name": "评价人姓名",
        "position": "职位",
        "company": "公司",
        "relationship": "关系",
        "date": "日期",
        "content": "评价内容"
      }
    }
  }
}
```

**注意**: 需要在 `en-US/`, `de-DE/`, `fr-FR/`, `ja-JP/` 目录下分别添加对应的翻译

---

#### 步骤 5: 添加UI翻译

**文件**: `languages/zh-CN/ui.json`

```json
{
  "testimonials": {

    "addTestimonial": "添加评价",
    "name": "评价人姓名",
    "position": "职位",
    "company": "公司",
```

---

#### 步骤 6: 修改模板HTML

**文件**: `templates/boss-zhipin/template.html`

在适当位置添加评价区块 (建议放在技能之后):

```html
{{#if testimonials}}
<section class="mb-8">
  <h2 class="text-2xl font-bold text-blue-600 mb-4 pb-2 border-b-2 border-blue-600">{{i18n.testimonials.section}}</h2>
  <div class="space-y-6">
    {{#each testimonials}}
    <div class="bg-gray-50 rounded-lg p-6 hover:shadow-md transition-shadow">
      <div class="flex justify-between items-start mb-2">
        <div>
          <h3 class="text-xl font-bold text-gray-800">{{name}}</h3>
          <p class="text-blue-600 font-medium">{{position}}</p>
          {{#if company}}<p class="text-gray-500 text-sm">{{company}}</p>{{/if}}
        </div>
        {{#if date}}<span class="text-sm text-gray-500 bg-blue-100 px-3 py-1 rounded-full">{{date}}</span>{{/if}}
      </div>
      {{#if relationship}}<p class="text-gray-500 text-sm mb-2">({{relationship}})</p>{{/if}}
      {{#if content}}<p class="text-gray-700 italic">"{{content}}"</p>{{/if}}
    </div>
    {{/each}}
  </div>
</section>
{{/if}}
```

**说明**: 使用 `{{#if testimonials}}` 判断模块是否有数据，没有则不显示

---

#### 步骤 7: 修改前端代码

**文件**: `public/app.js`

##### 7.1 添加模块导航按钮

在 `index.html` 的模块导航区域添加:

```html
<button data-module="testimonials" onclick="switchModule('testimonials')">Testimonials</button>
```

##### 7.2 处理新模块类型

在 `renderEditor()` 函数中添加条件判断 (约307行):

```javascript
if (currentModule === 'personal') {
  html += await renderPersonalEditor(data);
} else if (['experience', 'projects', 'education', 'certificates', 'testimonials'].includes(currentModule)) {
  html += await renderListEditor(currentModule, data);
} else if (currentModule === 'skills') {
  html += await renderSkillsEditor(data);
}
```

### 测试验证

#### 1. 配置加载测试
- 启动服务，检查是否识别新模块
- 查看模块导航是否显示"同事评价"按钮

#### 2. 数据操作测试
- 进入编辑模式
- 添加、编辑、删除评价记录
- 验证数据正确保存到 `testimonials.json`

#### 3. 多语言测试
- 切换不同语言
- 验证各语言版本数据独立存储
- 检查翻译显示是否正确

#### 4. 模板渲染测试
- 在不同模板下预览
- 检查评价区块布局是否正常
- 导出PDF/Word验证

## 完成后的效果

1. **模块导航**: 新增"同事评价"按钮
2. **编辑界面**: 自动生成表单，支持增删改查
3. **预览展示**: 按模板样式渲染评价内容
4. **多语言**: 支持各语言独立维护
5. **导出功能**: PDF/Word导出时包含评价内容

---

## 关键优势

- ✅ **零代码修改**: 只需编辑配置文件和数据文件
- ✅ **自动渲染**: 前端根据配置自动生成表单
- ✅ **配置驱动**: 符合系统设计架构
- ✅ **完全扩展**: 不影响现有功能
- ✅ **Schema验证**: 配置文件自动验证格式
