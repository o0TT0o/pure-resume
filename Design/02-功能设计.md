# 功能设计方案

## 核心功能

### 1. 简历展示（访客模式）
- **浏览简历**：以精美排版展示所有简历信息
- **响应式设计**：支持PC、平板、手机多端访问
- **打印预览**：提供打印友好的样式
- **分享链接**：生成简历分享链接
- **多语言支持**：
  - 右上角语言切换按钮（下拉菜单）
  - **配置驱动**：语言列表从 `languages/index.json` 动态读取
  - 语言状态显示（是否有内容）
  - 语言切换实时刷新页面
  - 语言状态保存在本地存储
  - ⚠️ **重要**：语言切换仅加载用户预先维护的对应语言数据，不提供翻译功能
- **模板切换**：
  - 支持多种模板风格（从 `templates/` 动态读取）
  - 实时预览模板效果
  - 模板切换保持数据完整性

### 2. 内容管理（管理员模式）
- **进入管理模式**：通过特定URL参数或隐藏入口进入
- **在线编辑**：直接在页面上编辑各模块内容
- **实时预览**：编辑时实时查看效果
- **拖拽排序**：工作经历、项目经验等可调整顺序
- **增删改查**：完整的CRUD操作
- **配置驱动的表单**：
  - 根据 `data/modules/config.json` 自动生成表单
  - 根据 `data/skills/config.json` 动态渲染技能分类
  - 根据 `data/personal/fields-config.json` 动态生成联系字段
- **多语言编辑**：
  - 支持在不同语言版本间切换编辑
  - 保存时自动写入对应语言的文件
  - 支持跨语言复制数据
  - 可直接编辑后台 JSON 文件

### 3. 导入导出功能

#### 导出为PDF
- 使用 `puppeteer` 或 `jsPDF` 生成PDF
- 支持A4纸张格式
- 保持页面排版一致性
- ⚠️ **重要**：导出当前用户所在的语言版本
- 导出选项：
  - 仅导出当前页面
  - 导出完整简历
  - 自定义页边距
  - 选择导出语言版本

#### 导出为Word
- 使用 `docx` 库生成Word文档
- 保持良好的文档结构
- 支持图片嵌入
- ⚠️ **重要**：导出当前用户所在的语言版本
- 导出选项：
  - 简版简历（一页）
  - 详细版简历（多页）
  - 自定义内容模块
  - 选择导出语言版本

#### 从PDF导入
- 使用 `pdf-parse` 解析PDF内容
- 智能识别简历结构
- 映射到对应数据字段
- 手动确认导入内容

#### 从Word导入
- 使用 `mammoth` 解析Word文档
- 保留文本格式和图片
- 智能内容识别
- 支持手动调整映射关系

### 4. 图片管理
- **上传头像**：支持拖拽上传，自动裁剪
- **项目图片**：支持多图上传，自动压缩
- **图片预览**：上传前预览效果
- **格式转换**：自动转换为WebP格式（可选）

### 5. 数据备份与恢复
- **自动备份**：每次保存自动备份
- **手动备份**：支持一键备份所有数据
- **备份列表**：查看所有历史备份
- **恢复功能**：从历史备份恢复数据
- **导出备份**：下载备份文件（ZIP格式）

### 6. 版本管理
- **历史版本**：记录每次修改
- **版本对比**：查看当前版本与修改后版本的差异
- **版本回滚**：恢复到任意历史版本
- **自动快照**：编辑前自动保存快照用于对比
- **修改高亮**：清晰展示新增、修改、删除的内容
- **单用户场景**：专为单用户本地系统设计，无需复杂的冲突处理

- 实时表单验证

     - 适合大多数岗位

## 技术实现

### 前端技术
- **基础框架**：纯JavaScript (ES6+)
- **UI框架**：**Tailwind CSS**（2026年主流方案）
- **模板系统**：Markdown + Tailwind CSS
- **Markdown解析**：`marked.js`
- **富文本编辑**：`Quill.js` 或 `TinyMCE`
- **日期选择**：`flatpickr`
- **图片处理**：`Cropper.js`
- **多语言支持**：`i18next` 或自定义实现（系统界面）
- **模板引擎**：`handlebars`（模板渲染）
- **版本对比**：`diff` 库（当前版本 vs 修改后）

### 后端技术
- **运行环境**：Node.js
- **Web框架**：Express.js
- **文件操作**：fs模块
- **导出PDF**：Puppeteer
- **导出Word**：Pandoc（命令行调用，格式最标准）+ docx库（备选）
- **导入PDF**：pdf-parse
- **导入Word**：mammoth
- **压缩备份**：archiver
- **Markdown解析**：marked
- **版本对比**：diff 库
- ⚠️ **注意**：不使用任何翻译服务、不使用任何第三方API、不使用暗黑模式

### 统一错误处理

#### 错误分类
```javascript
// 自定义错误类型
class AppError extends Error {
  constructor(message, code, statusCode, details = {}) {
    super(message);
    this.name = this.constructor.name;
    this.code = code;
    this.statusCode = statusCode;
    this.details = details;
    this.timestamp = new Date().toISOString();
  }

  toJSON() {
    return {
      name: this.name,
      message: this.message,
      code: this.code,
      statusCode: this.statusCode,
      details: this.details,
      timestamp: this.timestamp
    };
  }
}

// 配置错误
class ConfigError extends AppError {
  constructor(message, details = {}) {
    super(message, 'CONFIG_ERROR', 500, details);
  }
}

// 验证错误
class ValidationError extends AppError {
  constructor(message, details = {}) {
    super(message, 'VALIDATION_ERROR', 400, details);
  }
}

// 文件操作错误
class FileError extends AppError {
  constructor(message, details = {}) {
    super(message, 'FILE_ERROR', 500, details);
  }
}

// i18n 错误
class I18nError extends AppError {
  constructor(message, details = {}) {
    super(message, 'I18N_ERROR', 500, details);
  }
}
```

#### 全局错误处理中间件
```javascript
const logger = require('../utils/logger');
const { AppError } = require('../utils/errors');

function errorHandler(err, req, res, next) {
  // 记录错误日志
  logger.error('Error occurred:', {
    error: err.message,
    stack: err.stack,
    url: req.url,
    method: req.method,
    body: req.body,
    query: req.query,
    params: req.params,
    ip: req.ip,
    userAgent: req.get('user-agent')
  });

  // 判断是否为自定义错误
  if (err instanceof AppError) {
    return res.status(err.statusCode).json({
      error: {
        code: err.code,
        message: err.message,
        details: err.details,
        timestamp: err.timestamp
      }
    });
  }

  // 未知错误
  res.status(500).json({
    error: {
      code: 'INTERNAL_SERVER_ERROR',
      message: '服务器内部错误',
      timestamp: new Date().toISOString()
    }
  });
}

// 异步错误捕获包装器
function asyncHandler(fn) {
  return (req, res, next) => {
    Promise.resolve(fn(req, res, next)).catch(next);
  };
}
```

#### 错误监控服务
```javascript
class ErrorMonitor {
  constructor() {
    this.errorCounts = {};
    this.errorThresholds = {
      CONFIG_ERROR: { count: 5, timeWindow: 60000 }, // 1分钟内5次
      VALIDATION_ERROR: { count: 10, timeWindow: 60000 },
      INTERNAL_SERVER_ERROR: { count: 3, timeWindow: 60000 }
    };
  }

  // 记录错误
  recordError(error) {
    const errorCode = error.code || 'UNKNOWN';
    const now = Date.now();

    if (!this.errorCounts[errorCode]) {
      this.errorCounts[errorCode] = [];
    }

    this.errorCounts[errorCode].push(now);

    // 清理过期记录
    this.cleanupOldErrors(errorCode, now);

    // 检查是否超过阈值
    this.checkThreshold(errorCode);
  }

  // 检查错误阈值
  checkThreshold(errorCode) {
    const threshold = this.errorThresholds[errorCode];
    if (threshold) {
      const count = this.errorCounts[errorCode].length;
      if (count >= threshold.count) {
        this.sendAlert(errorCode, count);
      }
    }
  }

  // 发送告警
  sendAlert(errorCode, count) {
    logger.error('Error threshold exceeded', {
      errorCode,
      count,
      threshold: this.errorThresholds[errorCode].count,
      timeWindow: this.errorThresholds[errorCode].timeWindow,
      alert: 'HIGH_ERROR_RATE'
    });
  }
}
```

### 配置版本管理

```javascript
// 配置文件版本控制
class ConfigVersionManager {
  async getConfigVersion(configPath) {
    const config = await fs.readFile(configPath, 'utf8');
    const data = JSON.parse(config);
    return data.version || '1.0.0';
  }

  async updateConfig(configPath, updates) {
    const config = await fs.readFile(configPath, 'utf8');
    const data = JSON.parse(config);

    Object.assign(data, updates);
    data.updated = new Date().toISOString();
    data.version = this.incrementVersion(data.version);

    await fs.writeFile(configPath, JSON.stringify(data, null, 2), 'utf8');
  }

  incrementVersion(version) {
    const [major, minor, patch] = version.split('.').map(Number);
    return `${major}.${minor}.${patch + 1}`;
  }
}
```

## 多语言实现方案

### 语言切换机制
1. **界面切换**
   - 页面右上角语言选择下拉菜单
   - 显示语言图标和名称
   - 切换时保存到 localStorage

2. **数据加载**
   - ⚠️ **重要**：不提供任何翻译功能
   - 用户切换语言时，系统加载对应语言的简历数据
   - 如果该语言数据为空，显示提示信息
   - 所有数据由用户自行维护

3. **语言数据管理**
   - 系统界面语言：预设完整，无需维护
   - 简历内容数据：用户逐个语言版本编辑和维护
   - 支持直接编辑后台 JSON 文件
   - 支持跨语言复制数据

### 工作流程

#### 访客模式

2. 用户点击语言切换按钮，选择目标语言
3. 系统从 `data/[lang]/` 目录加载对应语言的简历数据
4. 如果数据为空，显示"该语言版本暂无内容"提示
5. 界面文本使用系统预设的对应语言包

#### 管理员模式
1. 管理员进入编辑模式
2. 系统加载当前语言的简历数据供编辑
3. 管理员编辑完成后，保存到对应语言的文件
4. 可以切换到其他语言版本继续编辑
5. 可以使用"复制数据"功能，将现有内容复制到新语言
6. 也可以直接编辑后台的 JSON 文件
